## 1. What this is

BetterCMS ("Content Canvas") is a **multi-workspace, multi-project CMS** in the space of Webflow + Sanity + Prismic, with a first-class AI agent. It is currently a **frontend-complete v1 skeleton**: every screen, flow, store and guard exists and works in the browser; all data is in-memory or localStorage (seeded demo data); Supabase powers real email auth only. Production work = swap the module stores for a backend keeping the same action APIs.

Product pillars:
1. **Structured content, visual editing** â€” developers define sections/schemas in code; marketers compose pages from approved sections; editors edit inline.
2. **An agent that does the busywork** â€” plans first, stages drafts, human approves, one-click undo, fully audited.
3. **AI-answer-engine delivery** â€” every page also serves as markdown (llms.txt, .md twins, content negotiation).
4. **Enterprise controls** â€” roles that cascade, custom roles with scoped permissions, workspace AI governance.

Live demo: https://bcms-demo.vercel.app (guest door: "Continue without signing in"; noindex enforced at header/meta/robots.txt level).

  - `agent/*` â€” see Â§7.

## 7. The Agent (flagship subsystem, `src/lib/agent/*` + `src/components/agent/*`)

**Trust model (non-negotiable product rules)**: the agent PLANS first â†’ a person APPROVES â†’ changes stage as DRAFTS (proposals) â†’ apply through real store actions â†’ **undo journal** enables one-click revert â†’ everything audited. Nothing auto-publishes. Ever.

**Run lifecycle** (`runs-store.ts`, timer-simulated streaming): `planning â†’ awaiting_approval â†’ applying â†’ review â†’ done | rejected`. Read-only skills (audit/aeo/links) skip review and emit findings. Store-level guards: `roleCanAct` (editor+ via effective role), `clampTier` (plan, then governance ceiling), governance skill/generator/BYOK checks â€” all return `""` when blocked so no code path bypasses UI gating.

**Skills** (`skills.ts`, 6): draft (needs collection), backfill SEO metadata, audit (read-only), internal links (read-only), AEO readiness (read-only, Max/Pro+), migrate-from-URL. `skillFromPrompt` routes free text (aeoâ†’migrateâ†’linksâ†’auditâ†’draftâ†’backfill priority). `simulate.ts` builds plans (goal/items/boundaries incl. brand-voice line when `hasBrandVoice`), schema-aware proposals, findings from real store data; `applyProposals` re-validates live state and returns truthful appliedIds + `UndoOp[]` (`removeEntry`/`removePage`/`restorePageField`); `revertRun` only removes still-draft items and only restores unchanged fields.

**Surfaces**: `/p/$project/agent` page (hero composer, skills gallery with credit/gate hints, Generators section, `AgentRoster` named agents w/ schedules + Run now, `AgentHistory` w/ search/CSV export/per-run Undo, `ConnectedAgents` dev-gated) â€˘ `AgentDock` 400px in-flow sidebar on every project screen (toggle in header, hidden on /agent and for reviewers; `dock-store` open/show(runId)) â€˘ workspace `/agent` (requires @project chip; hands off via `?run=`).

**Composer** (`AgentComposer.tsx`): contentEditable with manual DOM chip spans (`data-kind/id/label`, inline SVG icons), `@` mention groups (pages/collections/entries/files/project-unlock at workspace scope), `/` skills, file attachments, tier dropdown + **BYOK** ("Your models" from `byok-store`, provider key dialog; BYOK runs bill to the key, credits=0; hidden entirely when governance disallows).

**External agents** (`connected-store.ts` + Connected agents surfaces): MCP endpoint + `npx bettercms connect` setup command, client list (Claude Code/Cursor/VS Code/custom), per-project scoped grants with one-time token reveal + revoke; same guardrails (staging-only writes, audited).

## 15. Cross-cutting product rules (enforce in any rebuild)

1. Agent: plan â†’ human approval â†’ drafts only â†’ undo â†’ audit. Generators inherit all of it.
2. Publishing is always explicit and per-item; exactly one status indicator per screen.
3. Tiers are speed names (Lite/Balanced/Max); never expose model names except user-owned BYOK.
4. Usage/credit displays never use red; gates explain themselves ("Off in AI controls", "Available on Enterprise") and link to plans.
5. Copy: no em/en dashes anywhere; no hype verbs (unlock/seamless/empower); calm, specific, sentence case.
6. Locked features render as explanatory locked panels, never hidden mysteriously; role-gating hides tabs instead.
7. Every destructive/irreversible action gets a confirm or an undo path; one-time secrets block dismissal until copied.
8. The website preview canvas is always light, even in dark chrome.
